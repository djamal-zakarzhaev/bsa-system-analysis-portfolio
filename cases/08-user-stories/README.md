# Пользовательские истории

## Кратко

Для системы доставки сформулированы пользовательские истории менеджера, бухгалтера и курьера, затем подготовлен материал их согласования.

## Что демонстрирует

- выявление потребностей ролей;
- формулирование пользовательских историй;
- проверку полноты и непротиворечивости;
- фиксацию результатов согласования.

## Основные артефакты

- `artifacts/ex00_DLVR_us_manager.docx` — истории менеджера;
- `artifacts/ex01_DLVR_us_accountant.docx` — истории бухгалтера;
- `artifacts/ex01_DLVR_us_courier.docx` — истории курьера;
- `artifacts/ex02_DLVR_us.docx` — согласование.

## Как смотреть

Сравнить истории разных ролей и завершить просмотр документом согласования.

## Статус

Учебный кейс. Истории не являются подтверждённым backlog реального продукта.
