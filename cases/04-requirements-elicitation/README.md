# Выявление требований

## Кратко

Отработаны четыре метода выявления и согласования требований для системы доставки: ролевая игра, мозговой штурм, семинар и формирование видения.

## Что демонстрирует

- подготовку сессий выявления требований;
- фасилитацию обсуждений;
- фиксацию проблем, потребностей и решений;
- формирование видения продукта.

## Основные артефакты

- `artifacts/ex00_DMS_rolegame.docx` — ролевая игра;
- `artifacts/ex01_DMS_brainstorming.docx` — мозговой штурм;
- `artifacts/ex02_DMS_seminar.docx` — семинар;
- `artifacts/ex03_DMS_vision.docx` — видение проекта.

## Как смотреть

Открыть семинар и видение проекта; остальные документы показывают путь к результату.

## Статус

Учебный командный кейс. Материалы не заявляются как полностью индивидуальная работа.
