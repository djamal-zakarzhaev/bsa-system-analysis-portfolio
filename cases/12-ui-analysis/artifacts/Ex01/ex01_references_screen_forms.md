# Exercise 01 — Экранные формы классификаторов и справочников

Фрейм — это логическая область экранной формы: список записей, карточка выбранной записи, блок основных операций, блок поиска/фильтрации/сортировки и блок переходов.

## 1. Список классификаторов и справочников

| Идентификатор | Наименование | Тип | Связанные справочники / классификаторы, необходимые для заполнения | Для чего используется |
|---|---|---|---|---|
| REF‑01 | Филиалы | Справочник | Не требуется | Для хранения точек барбершопа, доступных клиенту при выборе места записи. |
| REF‑02 | Клиенты | Справочник | Каналы связи | Для хранения данных клиента, записи на услугу и отправки уведомлений. |
| REF‑03 | Сотрудники | Справочник | Роли сотрудников, Филиалы | Для хранения сотрудников, участвующих в обслуживании клиентов и администрировании системы. |
| REF‑04 | Роли сотрудников | Классификатор | Не требуется | Для определения прав и сценариев работы сотрудников в системе. |
| REF‑05 | Услуги | Справочник | Категории услуг | Для выбора услуги клиентом и настройки услуг, оказываемых мастерами. |
| REF‑06 | Категории услуг | Классификатор | Не требуется | Для группировки услуг, например: стрижка, уход, косметология. |
| REF‑07 | Услуги мастеров | Справочник | Сотрудники, Услуги, Роли сотрудников, Филиалы | Для определения, какие услуги оказывает конкретный мастер в конкретном филиале. |
| REF‑08 | Каналы связи | Классификатор | Не требуется | Для хранения каналов, через которые клиент может получать уведомления и сообщения. |
| REF‑09 | Правила уведомлений | Справочник | Каналы связи | Для настройки времени, канала и содержания автоматических уведомлений клиенту. |
| REF‑10 | Статусы записи | Классификатор | Не требуется | Для отображения состояния записи: создана, подтверждена, выполнена, отменена. |

## 2. Фреймы экранных форм справочников и классификаторов

| Идентификатор | Справочник / классификатор | Фрейм формы списка | Фрейм карточки записи | Фрейм основных операций | Фрейм обеспечивающих операций | Вызов операции, переход далее и отказ от операции |
|---|---|---|---|---|---|---|
| REF‑01 | Филиалы | Таблица филиалов: название, адрес, телефон, статус. | Карточка филиала: название, адрес, телефон, режим работы, статус. | Добавить, изменить, деактивировать, сохранить, отменить. | Поиск по названию и адресу, фильтр по статусу, сортировка по названию. | «Добавить» открывает пустую карточку; «Изменить» открывает выбранную карточку; «Сохранить» возвращает к списку; «Отменить» закрывает карточку без сохранения. |
| REF‑02 | Клиенты | Таблица клиентов: ФИО, телефон, канал связи, статус. | Карточка клиента: имя, телефон, предпочитаемый канал уведомлений, статус. | Добавить, изменить, деактивировать, сохранить, отменить. | Поиск по имени и телефону, фильтр по статусу и каналу, сортировка по имени. | После сохранения клиент доступен для записи на услугу; при отмене карточка закрывается без изменения данных. |
| REF‑03 | Сотрудники | Таблица сотрудников: ФИО, роль, филиал, статус. | Карточка сотрудника: ФИО, телефон, роль, филиал, статус. | Добавить, изменить, деактивировать, сохранить, отменить. | Поиск по ФИО и телефону, фильтр по роли, филиалу и статусу, сортировка по ФИО. | После сохранения сотрудник доступен для выбора в расписании и услугах мастеров; при отмене изменения не применяются. |
| REF‑04 | Роли сотрудников | Таблица ролей: код, название, описание, статус. | Карточка роли: код, название, описание, статус. | Добавить, изменить, деактивировать, сохранить, отменить. | Поиск по названию и коду, фильтр по статусу, сортировка по коду. | После сохранения роль доступна в карточке сотрудника; при отмене роль не создаётся или не изменяется. |
| REF‑05 | Услуги | Таблица услуг: название, категория, цена, статус. | Карточка услуги: название, категория, цена, длительность, описание, статус. | Добавить, изменить, дублировать, деактивировать, сохранить, отменить. | Поиск по названию, фильтр по категории и статусу, сортировка по названию и цене. | После сохранения услуга доступна для настройки мастерам и выбора клиентом; при отмене изменения не сохраняются. |
| REF‑06 | Категории услуг | Таблица категорий: код, название, статус. | Карточка категории: код, название, описание, статус. | Добавить, изменить, деактивировать, сохранить, отменить. | Поиск по названию, фильтр по статусу, сортировка по названию. | После сохранения категория доступна в карточке услуги; при отмене категория не создаётся или не изменяется. |
| REF‑07 | Услуги мастеров | Таблица связок: мастер, услуга, филиал, статус. | Карточка связки: мастер, услуга, филиал, период действия, статус. | Добавить, изменить, дублировать, деактивировать, сохранить, отменить. | Поиск по мастеру и услуге, фильтр по филиалу, услуге и статусу, сортировка по мастеру. | После сохранения клиент может выбрать указанного мастера для указанной услуги; при отмене связка не создаётся или не изменяется. |
| REF‑08 | Каналы связи | Таблица каналов: код, название, тип контакта, статус. | Карточка канала: код, название, тип контакта, описание, статус. | Добавить, изменить, деактивировать, сохранить, отменить. | Поиск по названию, фильтр по статусу и типу контакта, сортировка по названию. | После сохранения канал доступен в карточке клиента и правилах уведомлений; при отмене изменения не применяются. |
| REF‑09 | Правила уведомлений | Таблица правил: канал, время отправки, условие отправки, статус. | Карточка правила: канал, время до визита, шаблон сообщения, статус. | Добавить, изменить, дублировать, деактивировать, сохранить, отменить. | Поиск по каналу, фильтр по каналу и статусу, сортировка по времени отправки. | После сохранения правило используется системой при отправке уведомлений; при отмене правило не создаётся или не изменяется. |
| REF‑10 | Статусы записи | Таблица статусов: код, название, финальный статус, статус активности. | Карточка статуса: код, название, описание, признак финального статуса, статус активности. | Добавить, изменить, деактивировать, сохранить, отменить. | Поиск по названию и коду, фильтр по финальному статусу и активности, сортировка по коду. | После сохранения статус доступен в карточке записи; при отмене изменения не сохраняются. |

## 4. Основные операции и права доступа

Физическое удаление записей справочников не используется. Вместо удаления применяется деактивация: запись больше не выбирается в новых операциях, но остаётся в истории записей, расписаний, оплат, уведомлений и отзывов.

| Операция | Роли с доступом | Условия выполнения | Результат в справочнике / классификаторе | Результат в других частях системы |
|---|---|---|---|---|
| Просмотр списка | Менеджер — все справочники; мастер — только связанные с ним опубликованные данные; клиент — только опубликованные данные для записи | Пользователь имеет доступ к выбранному разделу. | Отображается список доступных записей. | Клиент видит доступные услуги и мастеров; мастер видит свои данные; менеджер управляет справочниками. |
| Просмотр карточки | Менеджер; мастер — только свои данные; клиент — только свои данные и опубликованные услуги/мастеров | Запись существует и доступна роли пользователя. | Открывается карточка выбранной записи. | Пользователь получает данные для дальнейшего действия: записи, редактирования или просмотра расписания. |
| Создание записи | Менеджер; клиент — только создание своей карточки при регистрации | Обязательные поля заполнены; уникальные значения не дублируются; связанные справочники заполнены активными значениями. | Создаётся новая активная запись. | Новая запись становится доступна в связанных процессах: бронировании, расписании, уведомлениях и выборе услуги. |
| Редактирование записи | Менеджер; клиент — только свои контактные данные; мастер — без права изменения административных справочников | Запись существует; пользователь имеет право на изменение; проверки пройдены. | Значения записи обновляются. | Изменения используются в записях, расписании, уведомлениях, отчётах и карточках пользователей. |
| Дублирование записи | Менеджер, только для справочников, где это указано | Исходная запись существует; пользователь меняет поля, которые должны быть уникальными. | Создаётся новая карточка на основе существующей записи. | Ускоряется создание похожих услуг, связок мастер–услуга и правил уведомлений. |
| Деактивация записи | Менеджер | Запись существует; деактивация не нарушает активные будущие записи. | Статус записи меняется на «Неактивна». | Запись скрывается из выбора для новых операций, но остаётся доступной в истории. |
| Отмена операции | Менеджер, клиент, мастер в доступных им формах | Пользователь нажал «Отменить» или «Назад» до сохранения. | Новые или изменённые данные не сохраняются. | Система возвращает пользователя к списку или предыдущему экрану без изменения данных. |

## 5. Результат операций по каждому справочнику

| Справочник / классификатор | Создание / изменение | Деактивация | Ограничения и условия |
|---|---|---|---|
| Филиалы | Филиал становится доступен для выбора при записи и настройке сотрудников. | Филиал скрывается из новых записей, но остаётся в истории. | Нельзя деактивировать филиал без проверки активных будущих записей. |
| Клиенты | Клиент может записываться на услуги и получать уведомления. | Клиент скрывается из активного выбора менеджера, история записей сохраняется. | Телефон должен быть уникальным. |
| Сотрудники | Сотрудник может быть назначен на роль и связан с филиалом. | Сотрудник скрывается из новых расписаний и записей. | Для участия в расписании и услугах мастеров у сотрудника должна быть подходящая роль. |
| Роли сотрудников | Роль доступна в карточке сотрудника. | Роль не используется для новых сотрудников. | Код роли должен быть уникальным. |
| Услуги | Услуга доступна для настройки мастерам и выбора клиентом. | Услуга скрывается из новой записи клиента. | Услуга должна иметь категорию и цену. |
| Категории услуг | Категория доступна в карточке услуги. | Категория скрывается при создании новых услуг. | Нельзя деактивировать категорию, если есть активные услуги без переноса в другую категорию. |
| Услуги мастеров | Клиент может выбрать мастера для конкретной услуги. | Связка мастер–услуга скрывается из новой записи. | Сотрудник и услуга должны быть активными. |
| Каналы связи | Канал доступен клиенту и правилам уведомлений. | Канал скрывается из новых настроек уведомлений. | Для клиента должен быть указан контакт, соответствующий типу канала. |
| Правила уведомлений | Система использует правило для автоматических уведомлений. | Правило больше не применяется к новым отправкам. | Должен быть выбран активный канал и указано время отправки. |
| Статусы записи | Статус доступен в карточке записи. | Статус скрывается из новых операций. | Код статуса должен быть уникальным; финальный статус должен быть явно отмечен. |

## 6. Обеспечивающие операции

| Справочник / классификатор | Сортировка: поля и состояние по умолчанию | Фильтрация: поля, сохранение фильтра, сообщение при отсутствии результата | Поиск: поля, точность совпадения, сообщение при отсутствии результата | Дублирование | Расчёт итогов |
|---|---|---|---|---|---|
| Филиалы | По умолчанию — по названию по возрастанию; дополнительно по статусу. | Статус; фильтр сохраняется до конца сессии; сообщение: «По выбранным условиям филиалы не найдены». | Название, адрес; частичное совпадение без учёта регистра; сообщение: «Филиалы не найдены». | Нет. | Количество филиалов, количество активных. |
| Клиенты | По умолчанию — по имени; дополнительно по дате регистрации. | Статус, канал связи; фильтр сохраняется до конца сессии; сообщение: «Клиенты по выбранным условиям не найдены». | Имя, телефон; частичное совпадение; сообщение: «Клиенты не найдены». | Нет. | Количество клиентов, количество активных. |
| Сотрудники | По умолчанию — по ФИО; дополнительно по роли и филиалу. | Роль, филиал, статус; фильтр сохраняется до конца сессии; сообщение: «Сотрудники по выбранным условиям не найдены». | ФИО, телефон; частичное совпадение; сообщение: «Сотрудники не найдены». | Нет. | Количество сотрудников, количество активных. |
| Роли сотрудников | По умолчанию — по коду роли. | Статус; фильтр сохраняется до конца сессии; сообщение: «Роли не найдены». | Код, название; точное совпадение по коду и частичное по названию; сообщение: «Роли не найдены». | Нет. | Количество ролей. |
| Услуги | По умолчанию — по названию; дополнительно по цене. | Категория, статус; фильтр сохраняется до конца сессии; сообщение: «Услуги по выбранным условиям не найдены». | Название, описание; частичное совпадение; сообщение: «Услуги не найдены». | Да, для похожей услуги. | Количество услуг, количество активных. |
