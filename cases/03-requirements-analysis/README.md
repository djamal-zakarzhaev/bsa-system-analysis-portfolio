# Анализ требований

## Кратко

Описаны текущее состояние сервиса доставки, его контекст, целевые изменения и входные/выходные потоки данных.

## Что демонстрирует

- анализ текущего и целевого состояния;
- построение контекстной диаграммы;
- выявление потоков данных;
- уточнение состава заинтересованных сторон.

## Основные артефакты

- `artifacts/ex00_dlv_as_is.docx` — текущее состояние и проблемы;
- `artifacts/ex01_dlv_context.png` — контекстная диаграмма;
- `artifacts/ex02_dlv_to_be.xlsx` — целевые изменения;
- `artifacts/ex03_dlv_streams.xlsx` — потоки данных;
- `artifacts/ex04_dlv_stakeholders.xlsx` — актуализированные стейкхолдеры и глоссарий.

## Как смотреть

Начать с контекстной диаграммы, затем сравнить текущее и целевое состояния.

## Статус

Учебный аналитический кейс, не описание действующей системы.
