# Exercise 00 — Выбор формы представления продукта

## Форма представления продукта

| Параметр | Описание | Обоснование для проекта |
|---|---|---|
| Устройства использования | Мобильный телефон пользователя: смартфон на iOS или Android. Основной размер экрана — экран мобильного устройства с вертикальной ориентацией. | Клиенту удобно записываться на услугу с личного телефона в любое время: выбрать филиал, услугу, мастера и свободный слот. Мастер также может быстро просмотреть своё расписание и записи с телефона. |
| Тип программного продукта | Адаптивное веб-приложение / mobile-first веб-интерфейс для онлайн-записи в барбершоп | Интерфейс доступен через сайт, но оптимизирован под мобильный экран, так как клиент чаще записывается со смартфона. Такой формат соответствует задаче онлайн-записи через сайт и сохраняет удобство мобильного сценария: клиенту не нужно находиться за компьютером, запись можно выполнить сразу после возникновения потребности. |
| Способы управления | Сенсорное управление: касания, прокрутка, выбор элементов из списков, нажатие кнопок, ввод текста с экранной клавиатуры. Дополнительно используется экранная клавиатура телефона для ввода имени, телефона, пароля и текста отзыва. | Основные действия пользователя в мобильном приложении — выбрать значение из списка, нажать кнопку подтверждения, ввести контактные данные и оставить отзыв. Для этого достаточно сенсорного экрана и экранной клавиатуры. |
| Потребности локализации | Основной язык интерфейса — русский. Формат телефона — российский номер телефона +7. Формат даты — ДД.ММ.ГГГГ. Формат времени — ЧЧ:ММ. Часовой пояс определяется по выбранному филиалу барбершопа. Каналы связи для уведомлений: Telegram, WhatsApp, VK, SMS. | Система используется клиентами и сотрудниками сети барбершопов. Для корректной записи важно одинаково отображать дату, время, телефон и канал связи. Часовой пояс нужен, если сеть работает в нескольких городах или филиалах. |
| Специальные возможности / особенности доступности | Интерфейс должен поддерживать читаемый размер шрифта, достаточный контраст текста и кнопок, крупные области нажатия, понятные сообщения об ошибках, возможность работы при слабом освещении. | Запись может выполняться с мобильного телефона в разных условиях: на улице, в транспорте, при плохом освещении. Крупные элементы и понятные ошибки снижают риск неправильного выбора услуги, мастера или времени. |
